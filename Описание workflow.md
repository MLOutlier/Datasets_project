**🎯 Общее назначение платформы**

Платформа предназначена для организации полного цикла подготовки данных
для обучения ИИ-моделей. Она позволяет заказчикам загружать сырые
данные, настраивать логику разметки, распределять задачи между
исполнителями, контролировать качество и получать готовый датасет в
стандартных форматах.

**Ключевая особенность:** платформа заточена под работу с закрытыми
данными (медицина, БПЛА, персональные данные) и образовательными
командами (студенты), где важны безопасность, контроль качества и
мотивация исполнителей.

Нужно, чтобы на платформе был реализован полный процесс от загрузки
датасета, до его разметки и возможности выгрузки.

Описание необходимого workflow:

**ЭТАП 1: Создание проекта заказчиком.**

1)  Создание проекта и загрузка датасета:

Заказчик создает проект и загружает датасет (сейчас есть возможность
загрузки фото и видео, пока что оставим только эти форматы).

2)  Предпросмотр и первичная обработка:

Перед окончательной загрузкой система показывает заказчику, как будут
обработаны данные: определяется формат, извлекаются элементы, удаляются
пустые записи и дубликаты, показывается preview. Это нужно, чтобы
заказчик понимал: правильно ли система распознала данные, какие записи
реально попадут в разметку, не сломался ли импорт.

3)  Настройка логики разметки:

Заказчик определяет какой тип разметки ему нужен: классификация,
бинарная классификация, детекция, сегментация и так далее. Но сейчас
главное сделать разметку видео и изображений, чтобы можно было выделять
объекты на изображениях. Еще заказчик указывает классы объектов с
цветами и описанием. Сейчас можно создать несколько тегов, но с
одинаковым цветом и плохим интерфейсом, это нужно исправить.

Еще заказчик настраивает:

-   Включена ли авто-разметка ИИ (и какая модель используется)

-   Кому выдавать задачи: всем, только специалистам, конкретной группе
    студентов

-   Параметры межкадровой обработки (для видео): интервал ключевых
    кадров, алгоритмы трекинга

4)  Прикрепление инструкции.

Заказчик создает или загружает инструкцию для исполнителей.

5)  Предобработка видео (если загружено видео):

> \- Кадры без целевых объектов фильтруются с помощью легкой ИИ-модели
>
> \- Выбираются ключевые кадры для разметки (каждый N-ный или по
> изменению сцены)
>
> \- Сохраняются метаданные: временная метка, номер кадра, источник

**ЭТАП 2: Автоматическая предразметка с помощью ИИ**

1.  **Запуск модели предразметки**

    -   Система применяет модель Segment Anything Model (SAM) или другую
        предобученную модель

    -   Модель автоматически предлагает ограничивающие рамки (bounding
        boxes) для объектов на кадрах

    -   Предложения сопровождаются оценкой уверенности (confidence
        score)

2.  **Фильтрация предложений ИИ**

    -   Предложения с уверенностью ниже порога (например, 0.7)
        отбрасываются

    -   Оставшиеся предложения помечаются как \"предварительные\" и
        требуют проверки человеком

3.  **Подготовка задач для исполнителей**

    -   Кадры с предложениями ИИ группируются в задания по 10 кадров

    -   Каждый кадр в задании входит в последовательность из не менее 3
        последовательных кадров (для межкадровой валидации)

    -   Задания сохраняются в очереди на распределение

**ЭТАП 3: Распределение задач между исполнителями**

1.  **Формирование пула исполнителей**

    -   Исполнители регистрируются в системе, указывают специализацию и
        опыт.

    -   Заказчик может массово добавить студентов из CSV-файла (ФИО,
        email, группа (уникальный идентификатор группы людей), роль)

    -   Система автоматически создает аккаунты и отправляет приглашения

2.  **Алгоритм автоматического распределения**

    -   Система фильтрует исполнителей по критериям: специализация,
        минимальный рейтинг, текущая нагрузка

    -   Из подходящих выбирается исполнитель с наименьшим количеством
        активных задач

    -   Задача назначается, исполнитель получает уведомление

3.  **Ручное распределение (опционально)**

    -   Заказчик может вручную назначить задачи конкретным исполнителям
        или группам. (Или открыть публичный доступ.)

    -   Полезно для сложных задач, требующих экспертных знаний

**ЭТАП 4: Разметка данных исполнителем**

1.  **Интерфейс выполнения задачи**

    -   У исполнителя должна быть вкладка со всеми доступными проектами
        и отдельная вкладка с проектами, в которые его записал заказчик
        или он сам записался (то есть, если начал размечать какой-то
        проект, то он помечается как активный и попадает в эту вкладку).

    -   На вкладке с проектами, исполнитель может кликнуть на проект,
        чтобы его открыть, там он может прочитать инструкцию и перейти к
        разметке.

    -   Исполнитель видит кадр с предложением ИИ (если включено)

    -   Может принять предложение, отредактировать рамку или нарисовать
        новую или несколько рамок, если требуется.

    -   Доступна инструкция, примеры, подсказки по спорным случаям

    -   Интерфейс поддерживает рисование прямоугольников, выбор класса,
        добавление комментариев, увеличение изображения

    -   Так как задачи будут автоматически распределяться, то
        исполнитель при входе в разметку должен последовательно
        выполнять свои задачи, переходя от одной к другой одним нажатием
        кнопки.

2.  **Сохранение результата**

    -   После завершения разметки кадра исполнитель нажимает
        \"Сохранить\" или \"Далее\"

    -   Система фиксирует: координаты рамки, класс, время выполнения,
        автора

    -   Если задание не завершено, сохраняется прогресс для продолжения
        позже

3.  **Обратная связь в реальном времени**

    -   ИИ-агент может сразу проверить разметку на очевидные ошибки
        (рамка за пределами кадра, пропущенный объект)

    -   При обнаружении проблемы исполнитель получает предупреждение и
        может исправить до отправки

**ЭТАП 5: Валидация и контроль качества**

1.  **Перекрестная проверка**

    -   Одно и то же задание отправляется на разметку нескольким
        исполнителям (по настройке, обычно 5)

    -   Для валидации: каждое задание проверяется 5 разными валидаторами

2.  **Контрольные вопросы (Golden Questions)**

    -   В каждое задание валидатора вставляется 10 контрольных кадров с
        заранее известной правильной разметкой

    -   Нужно предусмотреть хранение в системе уникальных контрольных
        вопросов с уже известным решением

    -   Если валидатор ошибается на контрольных более чем в 20% случаев
        --- задание отклоняется, исполнитель получает предупреждение

3.  **Автоматическая приемка**

    -   Задание принимается автоматически, если валидатор дал ≥80%
        правильных ответов на контрольные вопросы

    -   При более низкой точности задание возвращается на доработку

4.  **Межкадровая валидация (для видео)**

    -   Система автоматически проверяет согласованность разметки между
        соседними кадрами

    -   Используются алгоритмы трекинга: BOOSTING, MIL, KCF, CSRT,
        MedianFlow, TLD, MOSSE

    -   Если рамка, спроецированная из соседнего кадра, не пересекается
        с текущей (IoU \< 0.3) --- кадр помечается на повторную проверку

5.  **Межкадровая интерполяция (для ускорения)**

    -   Размечаются только ключевые кадры (например, каждый 10-й)

    -   Промежуточные кадры заполняются автоматически по формуле
        линейной интерполяции

    -   Валидатор проверяет интерполированные кадры; при низкой точности
        кадр отправляется на ручную разметку

**ЭТАП 6: ИИ-агенты в системе контроля качества**

1.  **ИИ как помощник контроля качества**

    -   Анализирует скорость выполнения: если кадр размечен менее чем за
        2 секунды --- флаг на проверку

    -   Ищет шаблонные ответы: если исполнитель ставит одинаковые рамки
        подряд --- предупреждение

    -   Сравнивает с другими аннотаторами

    -   Проверяет соответствие инструкции с помощью LLM: если описание
        не соответствует классу --- предупреждение

2.  **ИИ как агент управления процессом**

    -   Анализирует прогресс проекта и рекомендует действия: усилить
        инструктаж, привлечь больше исполнителей, упростить задачу

    -   Выявляет исполнителей с низким качеством и автоматически
        отправляет их на переобучение

    -   Предлагает, какие данные еще нужно собрать для улучшения
        репрезентативности датасета

3.  **Мониторинг безопасности (интеграция с Hive Trace, garak, NeMo
    Guardrails)**

    -   Отслеживает аномалии в поведении исполнителей и ИИ-агентов

    -   Обнаруживает попытки извлечения чувствительной информации или
        инъекции промптов

    -   Автоматически очищает данные от персональных данных перед
        выдачей задания (размытие лиц, удаление метаданных)

    -   Оповещает администратора о нарушениях через выбранные каналы
        связи

**ЭТАП 7: Мониторинг прогресса (Дашборд заказчика)**

Заказчик видит в реальном времени:

-   **Статистика по кадрам**: всего / размечено / валидировано /
    отклонено

-   **Метрики качества**: средняя точность аннотаторов, согласие между
    валидаторами, уровень брака

-   **Прогресс исполнителей**: кто работает хорошо, кто требует
    внимания, топ-исполнители

-   **Спорные случаи**: задачи с низким согласием, ожидающие эксперта

-   **Прогноз завершения**: оценка времени до готовности датасета на
    основе текущей скорости

**ЭТАП 8: Экспорт готового датасета**

1.  **Выбор формата экспорта**

    -   COCO JSON --- стандарт для задач детекции объектов

    -   YOLO TXT --- для обучения моделей семейства YOLO

    -   Pascal VOC XML --- совместимость с классическими фреймворками

    -   CSV с метаданными --- для табличного анализа

2.  **Структура экспортируемых данных**

    -   Папка с изображениями (разделена на train/val)

    -   Файлы аннотаций в выбранном формате

    -   Файл конфигурации (например, data.yaml для YOLO)

    -   Отчет о качестве (quality_report.json)

3.  **Отчет о качестве (Quality Report)**

    -   Общее количество кадров: размечено, валидировано, отклонено

    -   Процент брака и итоговое количество пригодных кадров

    -   Средняя точность аннотаторов и валидаторов

    -   Уровень согласия между аннотаторами (Inter-Annotator Agreement)

    -   Использовалась ли интерполяция и ее точность

    -   Оценка стоимости (если применимо)

**📦 Технические компоненты (концептуально)**

1.  **Backend (серверная часть)**

    -   Управление пользователями, проектами, задачами

    -   Обработка видео (разбиение на кадры, фильтрация)

    -   Интеграция с ИИ-моделями (SAM, YOLO, LLM)

    -   Асинхронная обработка задач (очереди, фоновые процессы)

2.  **Frontend (клиентская часть)**

    -   Интерфейс для заказчика: создание проекта, настройка, дашборд

    -   Интерфейс для исполнителя: разметка, прогресс, рейтинг

    -   Интерфейс для валидатора/эксперта: проверка, спорные случаи

    -   Адаптивный дизайн для работы на разных устройствах

3.  **Хранилище данных**

    -   База данных для метаданных: пользователи, проекты, задачи,
        аннотации

    -   Файловое хранилище для изображений и видео (локальное или
        S3-совместимое)

    -   Кеширование для ускорения доступа к часто используемым данным

4.  **Интеграции**

    -   ИИ-модели: локальные или через API (Hugging Face, NVIDIA NIM)

    -   Системы мониторинга: Hive Trace, garak, NeMo Guardrails

    -   Учебные платформы: экспорт оценок в Moodle, Canvas через LTI

    -   Уведомления: email, Telegram, веб-пуши

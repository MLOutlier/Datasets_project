# Работа с видео (дроны и другие объекты)

## Текущие возможности

### ✅ Что работает:

1. **Загрузка видео** до **500 МБ**
   - Форматы: MP4, AVI, MOV, MKV, WEBM, FLV
   - Автоматическое разбиение на кадры

2. **Извлечение кадров**
   - **С ffmpeg**: 1 кадр в секунду (настраивается)
   - **Без ffmpeg**: 1 фрейм (первый кадр)
   - Автоматическое создание задач для каждого кадра

3. **Разметка**
   - Каждый кадр = отдельная задача
   - Рисование bounding boxes (рамок вокруг дронов)
   - Переход к следующему кадру автоматически

---

## Как загрузить видео с дроном

### Шаг 1: Создать проект
```
http://localhost:3000/projects/create
```

- Тип: **Computer Vision**
- Название: "Drone Detection Video"
- Тип аннотации: **Bounding Boxes**
- Загрузите видео файл (до 500 МБ)
- Нажмите **Create Project**

### Шаг 2: Система обработит видео

При загрузке видео:
1. Файл сохраняется в `backend/media/uploads/{project_id}/`
2. Система извлекает кадры:
   - **Если установлен ffmpeg**: 1 кадр/сек
   - **Без ffmpeg**: первый кадр
3. Для каждого кадра создается задача

### Шаг 3: Начать разметку
```
http://localhost:3000/projects/{PROJECT_ID}/annotation
```

- Система покажет первый кадр
- Нарисуйте рамки вокруг дронов
- Нажмите **Submit Annotation**
- Переход к следующему кадру автоматически

## Настройка частоты кадров

Если нужно больше/меньше кадров в секунду:

### Изменить в `frames.py`:

```python
# backend/apps/cv_annotation/services/frames.py

# Строка 60 - изменить fps
cmd = [
    "ffmpeg",
    "-i", str(video_path),
    "-vf", "fps=1",  # ← Изменить на нужное значение
    # fps=0.5 = 1 кадр каждые 2 секунды
    # fps=1   = 1 кадр в секунду
    # fps=5   = 5 кадров в секунду
    "-q:v", "2",
    "-y",
    output_pattern
]
```

**Примеры:**
| fps | 1 мин видео | 10 мин видео |
|-----|-------------|--------------|
| 0.5 | 30 кадров | 300 кадров |
| 1   | 60 кадров | 600 кадров |
| 5   | 300 кадров | 3000 кадров |

## Оптимизация для больших видео

### Проблема: 400 МБ видео = много кадров

**Решения:**

1. **Уменьшить fps** (например, 0.5 = 1 кадр каждые 2 сек)
2. **Извлекать кадры только при изменении сцены** (требует доп. разработки)
3. **Разметить только первые N кадров** (выборочно)

### Добавить выборочную разметку:

В `ProjectDetailPage` можно добавить фильтр:
- Показать только первые 100 кадров
- Или только каждый 10-й кадр

## Установка ffmpeg

### Windows:
```powershell
# Через Chocolatey
choco install ffmpeg

# Или скачать с https://ffmpeg.org/download.html
```

### Проверить установку:
```bash
ffmpeg -version
```

### Без ffmpeg:
- Видео будет показан только **первый кадр**
- Все равно можно разметить его
- Для полного разбиения нужен ffmpeg

## Структура данных для видео

```
Проект "Drone Detection"
  └── Видео "flight_01.mp4" (400 MB)
       └── Кадры (извлеченные):
            ├── frame_0001.jpg (задача 1)
            ├── frame_0002.jpg (задача 2)
            ├── frame_0003.jpg (задача 3)
            └── ... (зависит от fps)
```

**Каждый кадр** = отдельная задача для разметки

## Пример workflow для дрона

### Новый рабочий 4-step процесс

1. **Выделение интервалов видео с объектом (hybrid):**
   - Откройте `Проект -> Этап 1-2: Интервалы`.
   - Выберите video asset.
   - Нажмите `Автогенерация черновиков`.
   - При необходимости добавьте интервалы вручную (start/end кадры).

2. **Валидация интервалов:**
   - На том же экране проверьте черновые интервалы.
   - Нажмите `Подтвердить draft интервалы` (или отклоните невалидные).
   - В bbox-пайплайн попадут только `approved` интервалы.

3. **Выделение объекта на изображении (bbox):**
   - Выполните `Finalize import` в проекте.
   - Аннотаторы получают assignments только по кадрам из `approved` интервалов.
   - Разметка делается в `/labeling/assignments/:assignmentId`.

4. **Валидация выделенного объекта:**
   - Customer/Admin: batch-валидация в `/quality`.
   - Reviewer/Admin: спорные кейсы по low-agreement в `/reviewer`.
   - После резолюции reviewer кадры переходят в финальный экспорт.

### E2E smoke-check (API+UI)

- Создайте CV-проект с `annotation_type=bbox`.
- Загрузите 1 видео в проект.
- Проверьте, что `ffmpeg` диагностируется в preview импорта.
- Создайте/провалидируйте интервалы и переведите часть в `approved`.
- Сделайте `Finalize import` и убедитесь, что появились задания.
- Отправьте минимум 2-3 аннотации и проверьте:
  - валидацию batch в `/quality`,
  - появление споров в `/reviewer` при низком agreement.
- Выполните экспорт `COCO/YOLO/VOC/CSV` и `download zip`.

### 1. Заказчик:
- Создает проект "Drone Detection"
- Загружает видео с дроном (400 МБ, 10 минут)
- Система извлекает 600 кадров (при fps=1)

### 2. Аннотатор:
- Открывает страницу разметки
- Видит кадр 1/600
- Рисует рамки вокруг всех дронов на кадре
- Нажимает Submit
- Переходит к кадру 2/600
- Повторяет для всех кадров

### 3. Результат:
- 600 размеченных кадров
- Координаты дронов на каждом кадре
- Готовый датасет для обучения ML модели

## Troubleshooting

### Видео не загружается
```bash
# Проверьте размер файла (лимит 500MB)
ls -lh video.mp4

# Увеличьте лимит в settings.py если нужно
MAX_UPLOAD_SIZE = 1024 * 1024 * 1024  # 1GB
```

### Мало кадров из видео
```bash
# Проверьте что ffmpeg установлен
ffmpeg -version

# Проверьте логи backend
docker-compose logs web | grep -i ffmpeg
```

### Долгая загрузка большого видео
- 400 МБ видео может грузиться 1-5 минут (зависит от сети)
- Извлечение кадров: 10-60 секунд
- Прогресс показывается на странице проекта
